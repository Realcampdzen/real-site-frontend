# Чек-лист для проверки деплоя изображений и видео

## Проблема
Изображения и видео не отображаются на продакшн-сайте, хотя на localhost:3000 все работает.

## Проверка файлов в git
Все файлы должны быть в репозитории. Проверьте:
```bash
git ls-files public/works/ | grep -E "\.(jpg|mp4)$"
```

## Возможные причины и решения

### 1. Файлы не загружены на сервер
**Решение:** Убедитесь, что после `git pull` на сервере все файлы из папки `public/works/` присутствуют.

### 2. Неправильная настройка веб-сервера
Если используется nginx/apache, нужно настроить раздачу статических файлов:
- Проверьте, что папка `public/works/` доступна для чтения
- Убедитесь, что веб-сервер настроен на раздачу файлов из корня проекта

### 3. Проблема с кодировкой имен файлов
Файлы с кириллицей в именах используют URL-кодирование в путях.
Проверьте в консоли браузера (F12) ошибки 404 для файлов.

### 4. Кэш браузера
Очистите кэш браузера или используйте режим инкогнито.

### 5. Размер файлов
Некоторые видео файлы большие (до 39 МБ). Убедитесь, что:
- Git LFS настроен правильно
- На сервере достаточно места
- Нет ограничений на размер файлов

## Проверка на сервере
1. Подключитесь к серверу
2. Перейдите в папку проекта
3. Выполните `git pull origin main`
4. Проверьте наличие файлов: `ls -la public/works/`
5. Проверьте права доступа: `chmod -R 755 public/works/`
6. Перезапустите веб-сервер
